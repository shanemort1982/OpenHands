## Minimal adapter point proposal
- Write facts externally (POST /memory/write) when certain events occur: after successful CmdRunAction (e.g., test passes), after FileEditObservation, or upon SuccessObservation. The safest centralized point is ActionExecutionClient.send_action_for_execution right after an Observation is produced (we already hook here for tool-call logs).
- Search facts externally (POST /memory/search) during agent step planning: inject near where Agent produces next Action, e.g., just before agent.step() or right after, feeding retrieved facts as additional context via a MessageAction or by augmenting the Condenser input. A safe backend point without UI changes: AgentController._step, immediately before/after action = self.agent.step(self.state).

### Specific adapter locations
- Write adapter (fact writes): openhands/runtime/impl/action_execution/action_execution_client.py in send_action_for_execution(), after observation_from_dict() and before returning. Scope facts by repo/project/env using:
  - repo: derive from self.config.workspace_mount_path_in_sandbox or git config present in workspace.
  - project: self.sid (session id) + agent id.
  - env: runtime (docker/local), user_id, and optional branch name.
  This centralizes after every tool call and avoids agent code injection.
- Search adapter (retrieval): openhands/controller/agent_controller.py in _step(), before calling self.agent.step(self.state) (to prefetch and inject), or after receiving the Action to possibly refine next step. Prefer pre-step fetch to feed into LLM context via MessageAction added to EventStream marked hidden to not clutter UI.

### Scoping strategy
- repo: compute a stable repository identifier (e.g., git remote URL hash or folder path)
- project: combine repo id + branch name + agent id
- env: runtime (docker/local), user id, session id (sid)
